# MindGuard AI Architecture

MindGuard AI is intentionally backend-first. Every user message passes through a deterministic pipeline before any response is returned.

```txt
Frontend / API client
  -> FastAPI route
  -> Session store
  -> Safety interceptor
  -> Medical boundary engine
  -> Emotion adapter
  -> PHQ-9 deterministic scoring engine
  -> DSM-5-informed non-diagnostic domain profiler
  -> Response engine
  -> Decision trace
```

## Why this satisfies the assignment

- The PHQ-9 score is never generated by an LLM.
- Safety checks run before normal conversation.
- Clinical diagnosis requests are blocked with the required exact sentence.
- DSM-5 language is handled as symptom-domain profiling, not diagnosis.
- The project is custom Python/FastAPI code.
